# 1. EC2とは？

EC2（Elastic Compute Cloud）は、AWSが提供する仮想サーバーサービスです。

シンプルに言うと、インターネット上にサーバーを用意して、自由に使えるサービスです。

### 特徴

- 数分でサーバーを作成できる
- CPUやメモリなどのスペックを選べる
- 使った分だけ課金される

### 今回の構成（例）

```text
インスタンスタイプ：t4g.micro（推奨）
OS：Amazon Linux 2023（推奨）
```

※Amazon Linux 2も利用できますが、現在はAmazon Linux 2023が推奨されています。

# 2. EC2インスタンスタイプとは？

EC2には用途ごとに多くの種類があり、2026年時点で100種類以上存在します。

整理するには、以下の順番で見ると理解しやすいです。

```text
① 役割 → ② 世代 → ③ CPU種類 → ④ 機能拡張
```

---

## ① 役割（用途）

| 分類 | ファミリ | 用途 |
|------|----------|------|
| 汎用 | T / M | Web、API |
| CPU特化 | C | 計算処理 |
| メモリ特化 | R | DB |
| ストレージ特化 | D / I / H | データ保存 |
| GPU | P / G | AI・機械学習 |

---

## ② 世代（数字）

例：`M5 → M6 → M7`

| 世代 | 意味 |
|------|------|
| 4 | 古い |
| 5 | 安定 |
| 6 | 主流 |
| 7 | 最新 |

基本的に、新しい世代ほど性能とコスト効率が良くなります。

---

## ③ CPU種類

| 記号 | CPU | 特徴 |
|------|-----|------|
| i | Intel | 互換性・安定性が高い |
| a | AMD | 価格が安い |
| g | Graviton（ARM） | コスト効率が高い |

---

### CPU選択の目安

| 用途 | 推奨CPU | 理由 |
|------|----------|------|
| Webアプリ | Graviton | コストと性能のバランスが良い |
| 個人開発 | Graviton | 価格が安い |
| 企業システム | Intel | 互換性重視 |
| 特定ライブラリ依存 | Intel | 安全性 |
| コスト重視（x86） | AMD | 価格が安い |

---

### CPU別の特徴

#### Intel
- 互換性が高く、ほとんどのソフトが問題なく動く
- 価格はやや高め

#### AMD
- Intelより安い
- 性能も十分

#### Graviton
- 価格が安く、電力効率も良い
- ARMのため、一部ソフトが対応していない場合がある

---

## ④ 機能拡張

| 記号 | 内容 |
|------|------|
| n | ネットワーク強化 |
| d | ローカルSSD |
| i | Intel |
| zn | 高クロック |
| flex | コスト最適 |

---

## サイズ（性能）

### 小規模

| サイズ | 用途 |
|--------|------|
| nano | 超軽量 |
| micro | 最小構成 |
| small | 少し余裕 |
| medium | 安定 |

### 中〜大規模

| サイズ | 用途 |
|--------|------|
| large | 小規模サービス |
| xlarge | 中規模 |
| 2xlarge〜 | 高負荷 |

---

### microの実態

```text
CPU：1〜2 vCPU
メモリ：約1GB
```

#### メリット
- 価格が安い
- 個人開発に向いている

#### デメリット
- 同時アクセスには弱い

---

## 具体例

### t3.micro

```text
T → 汎用
3 → 第3世代
→ Intel
```

### m6g.large

```text
M → 汎用
6 → 新しい
g → Graviton
```

### c6in.xlarge

```text
C → CPU特化
6 → 新しい
i → Intel
n → ネットワーク強化
```

# 3. 私が選んだインスタンス

## 前提条件

- 個人開発（Webツール公開）
- FastAPI / Python
- コストを抑えたい
- 将来的にスケールしたい

---

## 結論

```text
第一候補：t4g.micro
第二候補：t3.micro
```

---

## 理由

### コスト

Gravitonは同スペック帯の中で価格が低く、長期運用に向いています。

---

### Web用途との相性

```text
FastAPI / Python / API → ARM対応済み
```

問題なく動作します。

---

### 将来性

AWSはGravitonの利用を推進しており、本番環境でも採用が増えています。  
経験としても価値があります。
